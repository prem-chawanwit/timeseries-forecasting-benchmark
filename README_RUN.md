# วิธีการใช้งานและปรับแต่งโปรเจกต์ (Quick Start)

โปรเจกต์นี้ถูกออกแบบมาให้เป็นระบบ Automation ที่จบในคำสั่งเดียว และเก็บประวัติทุกการรันไว้ในโฟลเดอร์ให้คุณแบบแยกส่วนกัน

## 🚀 1. วิธีรันทดสอบ (รันแบบง่ายสุด)
ไม่ต้องรันอะไรแยกกันเลย ให้ใช้คำสั่งนี้คำสั่งเดียว:
```bash
python run_experiment.py
```
> ระบบจะไปอ่าน Data ทำความสะอาด แบ่ง Fold เทรนโมเดล 3 ตัว และออก Report ให้จบในรันเดียว ผลลัพธ์จะไปอยู่ที่โฟลเดอร์ `experiments/run_<วันที่_เวลา>/`

## 🛠 2. ถ้าอยากจะปรับแต่ง ต้องไปแก้ที่ไฟล์ไหนบ้าง?

หากคุณจะทำ Lab ต่อ นี่คือจุดสำคัญที่คุณต้องเข้าไปแก้ไข (แก้เสร็จแล้วกดรัน `python run_experiment.py` ได้เลย):

### A. อยากเปลี่ยนรอบเวลาสุ่มข้อมูล หรือปรับขนาดข้อมูลภายใน Fold
👉 **แก้ที่:** `src/etl/process_and_split.py`
* **เปลี่ยนรอบเวลา (1 ชั่วโมง -> 1 วัน):** หาบรรทัด `df.resample('1h')`
* **ปรับขนาด Test Set วงนอก:** หาบรรทัด `test_size = int(len(df_resampled) * 0.15)`
* **ควบคุม N (ขนาดข้อมูล) ภายใน Fold:** 
  ใน TimeSeriesSplit (ประมาณบรรทัด 60) จะมีตัวแปรควบคุม N ให้แก้ได้:
  * `test_size_in_fold = 500` (ถ้าอยากฟิกซ์ให้ Val.csv มีขนาด 500 แถวเป๊ะๆ)
  * `max_train_size = 1000` (ถ้าอยากฟิกซ์ว่า Train.csv ต้องไม่เกิน 1000 แถวเสมอ)

### B. อยากให้โมเดลหยิบคอลัมน์ (Features) ไปใช้ไม่เหมือนกัน
👉 **แก้ที่:** `src/train/<ชื่อโมเดล>/train.py`
* ในไฟล์ `train.py` ของแต่ละโมเดล จะมีตัวแปรชื่อ `feature_cols` อยู่ (เช่น บรรทัดที่ 21)
* คุณสามารถเพิ่มลดรายชื่อคอลัมน์ในวงเล็บ `[...]` ได้อย่างอิสระ โมเดลไหนอยากใช้อะไรก็แก้ที่ไฟล์ของโมเดลนั้นได้เลย

### C. อยากปรับค่า Hyperparameters (ความฉลาดของโมเดล)
* **XGBoost:** 👉 ไปแก้ที่ `src/models/xgboost_model.py` (แก้ไขค่า `max_depth`, `n_estimators`, `learning_rate`)
* **LSTM:** 👉 
  * ไปแก้โครงสร้าง Layer ที่ `src/models/lstm_model.py`
  * ไปแก้จำนวน Sequence ย้อนหลัง (`seq_length`) และจำนวนรอบเทรน (`epochs`) ที่ `src/train/lstm/train.py`

## 📂 3. รันเสร็จแล้ว ไปดูอะไรตรงไหน?
เข้าไปที่ `experiments/run_<เวลาล่าสุด>/` คุณจะเจอ:
1. `data/2_etl/data_distribution.png` (ดูกราฟหน้าตาข้อมูล และจำนวนที่ใช้)
2. `data/3_kfold_splits/split_meta.json` (ดูจำนวน N ว่าถูกแบ่งไป Fold ละกี่แถว)
3. `benchmark_comparison.png` (กราฟแท่งเปรียบเทียบความแม่นยำ MSE, MAE ของทั้ง 3 โมเดล)
4. `timeseries_comparison.png` (กราฟเส้นโชว์การทำนายเทรนด์ทับของจริง)
5. `statistical_tests.csv` (ดูว่าโมเดลไหนเก่งกว่ากันอย่างมีนัยสำคัญทางสถิติ - t-test)
